How we detect?

Mutation Testing

Mutation testing is a technique to evaluate the quality of your tests by programmatically mutating and making a series of small modifications to your code so that the tests no longer pass.

Definition

If bugs are crimes

And your tests are the police

Mutations are fake crimes that let you see the police is doing their job

Let me explain this with a basic example.

To assess the quality of a given test,

mutants are executed against the input test

to see if the seeded faults can be detected.

More technical explanation

Mutation, Mutants, Mutafuck what?

Each mutated version: Mutant

Mutated program + failing tests: killed mutant

Mutated program + passed tests: escaped mutant

Modifies a program in small ways

measure all the things!

Test suites are measured by the percentage of mutants that they kill. New tests can be designed to kill additional mutants.

Other metric that code coverage MSI CI environment Set certain percentage, let a build fail if the percentage is not reached.

How does it modify?

Mutators

Example


public function foobar(int $number): int
{
    return ++$number; // original
}
            

public function foobar(int $number): int
{
    return --$number; // mutated version
}

Another example


public function foobar(?Bar $foo): string
{
    if (null === $foo) { // original
        return 'foo';
    }
    // ...
}
            

public function foobar(?Bar $foo): string
{
    if (null !== $foo) { // mutated version
        return 'foo';
    }
    // ...
}
