Mutation Testing

Mutation testing is a technique to evaluate the quality of your tests by programmatically mutating by making a series of small modifications to your code and ensuring that the tests no longer pass.

If bugs are crimes

And your tests are the police

Mutations are fake crimes that let you see the police is doing their job

So how it works?

To assess the quality of a given test set,
these mutants are executed against the input test set

WIP
  • Modifies a program in small ways
  • Each mutated version is called a Mutant
  • Mutated program has failing tests: killed mutant
  • Mutated program passed tests: escaped mutant

Test suites are measured by the percentage of mutants that they kill. New tests can be designed to kill additional mutants.
